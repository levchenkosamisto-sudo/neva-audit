## ОБЩИЕ ВОПРОСЫ (ВСЕ ТРИ АУДИТОРА ЕДИНОГЛАСНО)

### В-01 [ЕДИНОГЛАСНО]: Токен в открытом виде в промпте — критическая уязвимость
**Ответ архитектора:** Признано. Токен NEVA-WebAI-Auditors имеет права только на запись в папку output/ репо neva-audit. Решение для Круга 2: GitHub Actions OIDC — токен полностью исчезает из промпта. Реализуем в Этапе 2.
**Статус:** ПРИНЯТО — Ч-ДУМА-SEC-01.

### В-02 [ЕДИНОГЛАСНО]: Веб-ИИ не могут сами запускать Python — нужен посредник
**Ответ архитектора:** Протокол устроен так: веб-ИИ выполняют Python через встроенный интерпретатор (ChatGPT Advanced Data Analysis, Gemini Code Execution). Для ИИ без Code Execution — Playwright fallback. duma_playwright.py v1.0 готов. neva_github_watcher.py v2.3 — 17/17 PASS.
**Статус:** ПРИНЯТО — двухрежимная архитектура.

### В-03 [ЕДИНОГЛАСНО]: Нет тайм-аута и арбитража при бесконечном несогласии
**Ответ архитектора:** Тайм-аут Круга: 48 часов. Максимум кругов: 5. На Круге 6 — эскалация Директору. Арбитраж: Директор Серж, решение окончательное.
**Статус:** ПРИНЯТО — IEEE 1028 + CMMI совместимо.

## ВОПРОСЫ GPT-4o

### GPT-В-01: Нет формального контракта данных input/output
**Ответ:** Добавляем governance/duma/schema/ с JSON Schema: round_input_schema.json, round_output_schema.json, synthesis_schema.json. Реализуем в Этапе 2.
**Статус:** ПРИНЯТО.

### GPT-В-02: Нет автомата состояний процесса аудита
**Ответ:** Явный FSM в виде state/duma_fsm.json — машина состояний с текущим состоянием, историей переходов. Watcher читает и обновляет.
**Статус:** ПРИНЯТО.

### GPT-В-03: Нет формата synthesis/ файлов
**Ответ:** Добавляем template_synthesis.md в governance/duma/prompts/. Включает список вопросов по аудитору, консолидированный список, группировку по теме.
**Статус:** ПРИНЯТО.

### GPT-В-04: Нет правил консенсуса
**Ответ:** Правила: 3/3 СОГЛАСЕН → ЗАКРЫТ ✅. 2/3 → доп. круг (до 5). 1/3 или 0/3 → эскалация Директору. После 5 кругов → Директор решает финально.
**Статус:** ПРИНЯТО.

### GPT-В-05: Как Watcher определяет принадлежность к текущему кругу?
**Ответ:** По имени файла: round_{N}_{auditor}_{doc}.md. Watcher парсит N как номер круга. Текущий ожидаемый круг хранится в state/duma_fsm.json.
**Статус:** ПРИНЯТО.
