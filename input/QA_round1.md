## ОБЩИЕ ВОПРОСЫ (ВСЕ ТРИ АУДИТОРА ЕДИНОГЛАСНО)

### В-01 [ЕДИНОГЛАСНО]: Токен в открытом виде в промпте — критическая уязвимость

**Ответ архитектора:**
Признано. Это осознанный компромисс для текущего этапа: токен NEVA-WebAI-Auditors имеет права только на запись в папку output/ репо neva-audit. Он не даёт доступа ни к коду NEVA, ни к секретам, ни к продакшену. Аналог — публичный dropbox для сдачи работ.

Тем не менее уязвимость реальная: злоумышленник может флудить output/ мусором.

**Решение для Круга 2:**
Перевести на GitHub Actions workflow с OIDC: аудитор отправляет запрос через публичный API endpoint (без токена), GitHub Actions записывает в репо. Токен полностью исчезает из промпта. Реализуем в Этапе 2.

**Статус:** ПРИНЯТО — будет реализовано в Этапе 2 (Ч-ДУМА-SEC-01).

### В-02 [ЕДИНОГЛАСНО]: Веб-ИИ не могут сами запускать Python — нужен посредник

**Ответ архитектора:**
Правильное замечание. Протокол сознательно устроен так: веб-ИИ (ChatGPT, Gemini, DeepSeek в браузере) сами выполняют Python-скрипт через встроенный интерпретатор (ChatGPT Advanced Data Analysis, Gemini Code Execution). Это реально работает — Круг 1 это подтвердил: все 6 файлов залиты.

Для ИИ без Code Execution: Watcher и duma_playwright.py выступают посредниками.

**Текущий статус:** duma_playwright.py v1.0 создан и готов. neva_github_watcher.py v2.3 — 17/17 PASS.

**Статус:** ПРИНЯТО — архитектура двухрежимная.

### В-03 [ЕДИНОГЛАСНО]: Нет тайм-аута и арбитража при бесконечном несогласии

**Ответ архитектора:**
- Тайм-аут Круга: 48 часов с момента отправки ответов архитектора.
